This is a compilation of notes to be used as supplementary material for a gentoo linux install. Use alongside the handbook found at wiki.gentoo.org

Installation theme: https://www.youtube.com/watch?v=Z6hRV-Nq1Q0

1. Pre-installation

1.1 Prepare an installation medium

The easiest thing to do is boot into a live Linux iso of some kind with a graphical desktop all set up. This makes it nice and easy to connect to the internet and do everything you’ll have to do. Linux Mint is a good choice.

Download a fresh iso and use the dd utility to make a bootable live usb:

$ dd bs=4M if=path/to/archlinux.iso of=/dev/flashDrive status=progress; sync

Once you’re booted into a live environment, get set up and ready to install. Make sure you’re connected to the internet, pull up a terminal and a web browser and get comfy I usually disable pointer acceleration too and you might need to install vim depending on the live iso you’re using:

(for Mint and other Debian derivatives):

$ sudo apt install neovim

Once you’re ready to go, open the Gentoo Installation Alternatives page.

That page has special instructions that need to be followed at various points in the install process, so keep it up for reference.

In the terminal, switch to the root user so you don’t need to type sudo before every command:

$ sudo su

Then, create the /mnt/gentoo mount point:

# mkdir /mnt/gentoo

Begin following the handbook from the Disks page.

2. Preparing the disks

For the partition setup, use 3 partitions as follows:

partition filesystem size description
/dev/sdx1 fat32 1 GiB EFI (boot) partition
/dev/sdx2 swap RAM * 1.5 Swap partition
/dev/sdx3 xfs remainder root partition

For explanations of the setup or alternatives, consult the handbook page. Use cfdisk to make the partitions, it’s easier than fdisk. If cfdisk isn’t available for some reason, fdisk instructions are given in the handbook. After making the partitions, create the filesystems.

EFI partition gets formatted as fat32:

# mkfs.vfat -F 32 /dev/sdx1

Root partition as xfs:

# mkfs.xfs -c options=/usr/share/xfsprogs/mkfs/lts_6.6.conf /dev/sdx3

Activate the swap partition:

# mkswap /dev/sdx2
# swapon /dev/sdx2

Mount the partitions:

# mount /dev/sdx3 /mnt/gentoo
# mkdir /mnt/gentoo/efi
# mount /dev/sdx1 /mnt/gentoo/efi

3. The stage file

Set the current directory to the location of the mount used for the install:

# cd /mnt/gentoo

Download the stage tarball. In a browser, open the downloads page.

Select your architecture, probably amd64, and choose a stage file, probably ‘desktop profile | systemd’

Copy the URL for the stage file download and paste it into a wget command:

# wget <STAGE_FILE_URL>

The URL as of our last installation is here for convenience.

Extract the tar file with the following command:

Note that this command is slightly different from the one in the handbook, it’s to be used when installing from a live Linux environment:

# tar --numeric-owner --xattrs -xvJpf stage3-*.tar.xz -C /mnt/gentoo

Use an editor to set the initial make.conf variables:

"/mnt/gentoo/etc/portage/make.conf"
-----------------------------------
COMMON_FLAGS="-march=native"
RUSTFLAGS="${RUSTFLAGS} -C target-cpu=native"
MAKEOPTS="-jN -lN"

Note that These are flags to ADD to the variables. if the variables already have some default options, add these in addition to those already there. These are just the initial basic settings.

The values for -jN and -lN should be something that makes sense for the target system. Reference this page for more info.

4. Installing base system

Copy DNS info so the chrooted environment can still connect to the internet:

# cp --dereference /etc/resolv.conf /mnt/gentoo/etc/

Mount all necessary filesystems:

# mount -o bind /proc /mnt/gentoo/proc
# mount --rbind /sys /mnt/gentoo/sys
# mount --make-rslave /mnt/gentoo/sys
# mount --rbind /dev /mnt/gentoo/dev
# mount --make-rslave /mnt/gentoo/dev
# mount --bind /run /mnt/gentoo/run
# mount --make-slave /mnt/gentoo/run

Change root to the new system:

# chroot /mnt/gentoo /bin/env -i TERM=$TERM /bin/bash
# env-update
# source /etc/profile
# export PS1="(chroot) $PS1"

Pull the latest snapshot:

# emerge-webrsync

List any news items with portage:

# eselect news list

and read through them:

# eselect news read

Clear them when done:

# eselect news purge

Edit the make.conf file and set these variables. You’ll need to use nano just this once before vim is installed. These are flags to ADD to the variables. If the variables already have some default options, add these in addition to those already there.

"/etc/portage/make.conf"
------------------------
FEATURES="parallel-fetch getbinpkg"
EMERGE_DEFAULT_OPTS="--jobs=X --load-average=X --usepkg=y"
ACCEPT_LICENSE="*"

Set the PORTAGE_BINHOST variable to these URLs for downloading binary packages:

https://distfiles.gentoo.org/releases/amd64/binpackages/23.0/x86-64-v3/ \
https://distfiles.gentoo.org/releases/amd64/binpackages/23.0/x86-64/

Note: Maybe update this section to use the binrepos.conf file because the way we do it here is technically deprecated

Run this command so that binary packages can be used:

# getuto

Now update the mirrors with the mirrorlist tool. first install it:

# emerge -a --oneshot app-portage/mirrorselect

Run this command to select the 5 fastest mirrors:

# mirrorselect -s5 -o >> /etc/portage/make.conf

Install neovim so it’s easier to edit these files:

# emerge -a app-editors/neovim

Select the correct profile (should match the stage file used earlier):

# eselect profile list
# eselect profile set <CHOICE>

Set CPU specific flags:

# emerge -a --oneshot app-portage/cpuid2cpuflags
# echo "*/* $(cpuid2cpuflags)" > /etc/portage/package.use/00cpu-flags

Unset the default GPU flag and set an appropriate one. GPU_DRIVER_NAME will likely be one of nvidia, intel, or amdgpu radeonsi. consult the handbook for a comprehensive list.

"/etc/portage/package.use/00video_cards"
----------------------------------------
*/* VIDEO_CARDS: -* <GPU_DRIVER_NAME>

Update the system to account for any changes made so far:

# emerge -a --update --deep --changed-use @world

Remove and unused packages after updating:

# emerge -a --depclean

Set timezone:

# ln -sf ../usr/share/zoneinfo/America/New_York /etc/localtime

Set locales by un-commenting the relevant line in the locale.gen file:

"/etc/locale.gen"
-----------------
en_US

Generate locales:

# locale-gen

List the locales and then select the correct one:

# eselect locale list
# eselect locale set <CHOICE>

Reload environment:

# env-update && source /etc/profile && export PS1="(chroot) ${PS1}"

5. Configuring the kernel

Install linux firmware:

# emerge -a sys-kernel/linux-firmware
# emerge -a sys-firmware/sof-firmware

Install CPU microcode updates. AMD CPUs are included in the above firmware package. Intel CPUs require:

# emerge -a sys-firmware/intel-microcode

To choose the bootloader, create this file with the following flags:

"/etc/portage/package.use/systemd-boot"
---------------------------------------
sys-apps/systemd boot
sys-apps/systemd-utils boot kernel-install
sys-kernel/installkernel systemd systemd-boot

Then create the following file:

"/etc/kernel/cmdline"
---------------------
quiet splash

And symlink it with this other file:

# mkdir -p /etc/cmdline.d
# ln -s /etc/kernel/cmdline /etc/cmdline.d/00-installkernel.conf

Then install the installkernel:

# emerge -a sys-kernel/installkernel

Set this flag so the installkernel handles the initramfs with dracut:

"/etc/portage/package.use/installkernel"
----------------------------------------
sys-kernel/installkernel dracut

Some bootloaders need the root partition explicitly defined:

# mkdir /etc/dracut.conf.d
# blkid >> /etc/dracut.conf.d/00-installkernel.conf

Go into that new file and identify the root partition and set it up like so (note leading and trailing spaces before and after the UUID):

"/etc/dracut.conf.d/00-installkernel.conf"
kernel_cmdline=" root=UUID=XXXXXXXXX... "

Note: Do we need UKI? no idea

Install the distribution kernel:

# emerge -a sys-kernel/gentoo-kernel-bin

Clean up after install:

# emerge --depclean

Add this USE flag globally to make.conf so that the initramfs are automatically rebuilt when the kernel updgrades:

"/etc/portage/make.conf"
------------------------
USE="dist-kernel"

6. Configuring the system

Set up your fstab file. The handbook does this manually but genfstab is easy.

# emerge -a sys-fs/genfstab

Generate the fstab using UUID labels

# genfstab / -U > /etc/fstab

Set the hostname:

# echo <HOSTNAME> > /etc/hostname

Set up network manager by first adding the USE flag to make.conf:

"/etc/portage/make.conf"
------------------------
USE="networkmanager"

Then install the package:

# emerge -a net-misc/networkmanager

Finally, enable the service:

# systemctl enable NetworkManager

Edit the hosts file /etc/hosts.

7. Installing system tools

Install a file indexer to speed up file location:

# emerge -a sys-apps/mlocate

Install shell completion:

# emerge -a app-shells/bash-completion

Timesync service for the clock:

# systemctl enable systemd-timesyncd.service

Filesystem tools:

# emerge -a sys-fs/xfsprogs sys-fs/dosfstools

Recommended for correct scheduling behavior:

# emerge -a sys-block/io-scheduler-udev-rules

Portage tools:

# emerge -a app-portage/gentoolkit

8. Configuring the bootloader

Systemd-boot should already be installed

# bootctl install

Check for a new boot entry with this:

# bootctl list

Exit the chrooted environment and unmount all partitions:

# exit
# cd
# umount -l /mnt/gentoo/dev{/shm,/pts,}
# umount -R /mnt/gentoo

Finally, reboot. Be sure to remove the installation medium before restarting.
